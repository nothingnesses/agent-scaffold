This section is the single living human-decision queue for the task (the model adopted in `human-review-queue`). Each item has a stable id, a one-line ask, a status, and a context pointer into the step or ledger that carries the detail. Statuses are `open` (awaiting a human decision), `decided -> folded into <slug>` (resolved; the decision and reasoning live in that step, and the item stays here marked resolved so it is not re-addressed), or `superseded` (replaced by a later item, kept for provenance). The orchestrator maintains this queue under the push-at-checkpoint rule stated in the Documentation Protocol; a folded decision reopens only on evidence that beats its recorded reasoning.

OQ-K) elsewhere in this plan are provenance markers for questions resolved and folded before this queue existed, not live entries; a reference such as "from OQ-E" points to a decision now recorded inline in the relevant step.
