## Findings

### L1 (low) - The byte-identity invariant holds only under the formatter; the raw render leaves two trailing blank lines

Evidence: `pack/AGENTS.md` ends `...\n\n{{principles}}\n\n{{instrument}}\n`. With `--instrument` off, `{{instrument}}` substitutes to `""`, so the raw rendered tail is `<last principle>\n\n\n` (verified: `tail -5 <scratch>/off/AGENTS.md | cat -A` shows two trailing `$`-only lines that the committed `AGENTS.md` does not have; `diff <scratch>/off/AGENTS.md AGENTS.md` reports `114,115d113` = two extra blank lines). The committed dogfood output is byte-identical only because `just scaffold-self` runs `nix fmt` (prettier, `proseWrap=never`) afterwards, which collapses the trailing blanks; the `git diff --stat` check is consequently empty. This is exactly the formatter-dependency the review brief anticipated: the blank line between `{{principles}}` and `{{instrument}}` gives ON a correct Markdown separation, and the formatter cleans up the OFF residue. Impact: a downstream user who runs `agent-scaffold` without `--instrument` and without a Markdown formatter now gets an `AGENTS.md`/`.agents/AGENTS.reference.md` with two trailing blank lines that the pre-change binary did not emit, so the plan's literal "byte-identical to the non-instrumented run" (plan line 524) holds only after formatting, not for the raw tool output. Trailing-whitespace-only, semantically inert, and normalised by any Markdown formatter, hence low. If the project wants byte-identity without depending on a formatter, the placeholder placement would need restructuring so the OFF substitution collapses to a single trailing newline.

### L2 (low) - `unwrap_or_default()` also masks non-NotFound IO errors on the fragment read

Evidence: `src/main.rs` `build_assets`, `if instrument { source.read("instrument.md").unwrap_or_default() } else { String::new() }`. `PackSource::read` (`src/manifest.rs:179-193`) returns `io::Result`, so `unwrap_or_default()` swallows every error kind, not just `NotFound`: a present-but-unreadable `instrument.md` (permission denied) or invalid UTF-8 silently yields an empty instrumentation block even though `--instrument` was requested, with no warning (tension with Principles 12 "fail fast and loudly" and 15 "make failure and absence explicit"). This is identical to the pre-existing `pack_principles` pattern (`Err(_) => Ok(Vec::new())`), so it is consistent with the codebase convention and is not a defect introduced by this step; I raise it only so the triager can decide whether the shared read-swallowing convention is worth tightening (distinguishing `NotFound` -> empty from other errors -> fail). Low because it is a latent, codebase-wide pattern, not new, and the common case (genuine absence) is handled correctly.

### L3 (low) - Records carry no task or timestamp delimiter, which may limit the cross-task aggregation the design intends

Evidence: `pack/instrument.md` defines four record types whose only identifying field is `artifact`. The log at `docs/metrics/workflow.jsonl` "accumulates across tasks" and the design goal (plan line 522) is that "a small script aggregates many tasks." With no per-task marker (task id, or a timestamp), two separate tasks that each review an artifact named e.g. `plan` produce interleaved-by-append records that a script cannot cleanly segment by task; `consecutive_clean` resets give only a heuristic boundary. This is within the plan's enumerated fields (`workflow-calibration` "Data to gather" does not list a task id), so it is spec-compliant, but it is a completeness observation against the stated aggregation purpose and against `workflow-calibration`'s key signal ("a round following a clean round finds a valid issue on an unchanged artifact"), which needs rounds reliably grouped within one artifact's loop. Low; flagging for the deferred `workflow-calibration`/`state-schema` owners to consider a task/run delimiter.

### L4 (low) - No automated test locks the byte-identity KEY INVARIANT

Evidence: the new `instrument_off_omits_the_block_and_on_includes_it` test asserts only the presence/absence of the section heading and the placeholder string; it does not assert that the OFF output equals the pre-change bytes. The KEY INVARIANT (OFF byte-identical) is verified only manually via `just scaffold-self` + `git diff`, which depends on a human running it and on the formatter. A regression that reintroduced a stray blank line in the formatted OFF path (or changed the fragment placement) would not be caught by the unit suite. Low; the invariant is verified today and the golden `scaffold-self` sync is the project's chosen guard, but no unit test pins it.
