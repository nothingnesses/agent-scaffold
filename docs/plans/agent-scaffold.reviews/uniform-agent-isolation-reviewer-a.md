It directly contradicts the new uniform rule now in `pack/AGENTS.md:83,89,91` ("Run every spawned agent ... in the strongest isolation"). A user who scaffolds the isolation module gets `AGENTS.md` saying everyone isolates and `isolation-guidance.md` saying read-only agents need none, in the same drop. - Suggested fix: rewrite the last sentence of `pack/isolation-guidance.md:37` (and soften the writer-only framing at 30/37) to the uniform rule, per the pack `AGENTS.md` wording. Because a self-scaffolded `.agents/isolation-guidance.md` copy does not exist in this repo (this repo did not scaffold `--module isolation`), only the pack source needs the edit.

## Finding 2: AGENTS.md internally disagrees on whether an explorer is a writer or a read-only agent, and the authored prose contradicts the decision's own stated resolution

- Severity: high.
- Evidence:
  - `AGENTS.md:65` (Design explorations, unchanged): "Explorers are writer agents, so the file-safety and writer-isolation rules below apply to them." That section also says an exploration gets "a single light review pass (one reviewer, one triager ...)".
  - `AGENTS.md:83` (new): "the writers and the read-only reviewers, triager, and explorers alike".
  - `AGENTS.md:89` (new): "a read-only agent (a reviewer, the triager, or an explorer) is read-only with respect to that product, authoring only its own findings or notes, which inform the work but are never accepted as product, and that independence is what keeps review from having to review its own reviews."
  - `AGENTS.md:91` (new): "the read-only reviewers, triager, and explorers alike".
  - Same wording in `pack/AGENTS.md` and `.agents/AGENTS.reference.md` at the corresponding lines.
  - The decision receipt itself (`docs/plans/agent-scaffold.md:167` Q-61 entry; `docs/plans/agent-scaffold.plan.toml` Q-61 `ask`): "RESOLVES the explorer-classification inconsistency (explorers are writers that isolate, consistent with the 'Explorers are writer agents' line, so they are no longer listed among read-only agents needing no worktree)." The step-detail rationale (`docs/plans/agent-scaffold.md:901`) repeats: "explorers were already called writer agents that isolate".
- Why it matters: the change's stated resolution is "explorers ARE writers; stop listing them as read-only." The prose it actually wrote does the opposite: lines 83/89/91 group explorers with "read-only reviewers, triager, and explorers" and explicitly call an explorer "a read-only agent". So (a) the implemented text contradicts its own decision receipt, and (b) `AGENTS.md` now asserts both "Explorers are writer agents" (65) and "an explorer ... is a read-only agent" (89) in one shipped document. The change claims to have removed the explorer contradiction but instead relocated and, arguably, deepened it. The justification given at line 89 ("never accepted as product", "keeps review from having to review its own reviews") is further undercut by line 65, which says an exploration IS reviewed by a reviewer and triager and IS synthesised into the decision, so the "read-only, output never reviewed" rationale does not fit explorers.
- Note on the roles anchor: `AGENTS.md:25` ("the planner and the implementer are writers ... and the reviewers and the triager are read-only ...") does not mention explorers at all and states "'Writer agent' below means a spawned writer role." It coheres with the new "read-only w.r.t. plan and code" framing for reviewers/triager, but its silence on explorers leaves the writer-vs-read-only question for explorers unresolved at the very place that defines the terms, which is part of the same inconsistency.
- Suggested fix: pick one classification for explorers and apply it everywhere. The decision receipt says "writers", which also matches line 65 and the "explorations get a review pass" mechanics; if that is the intent, drop "explorers" from the "read-only reviewers, triager, and explorers" groupings at 83/89/91 and instead state that explorers isolate as writers (their exploration is reviewed writer output). If instead the intent is genuinely to reclassify explorers as read-only, then line 65 ("Explorers are writer agents"), the optional review-pass sentence, and the Q-61 receipt text all need reconciling, because they currently say the opposite.
