### `agents-md-drift-guard`: whole-file normalized drift guard for the committed AGENTS.md (`Q-64`)

Assert that the committed root AGENTS.md and its `.agents/AGENTS.reference.md` copy match a fresh `build_assets` render, the `render --check` equivalent for AGENTS.md. Before this, only per-fragment `.contains()` guards existed; nothing asserted that a fresh render matches the whole committed file, so a hand-edit of authored prose or a dropped generated slot was not caught the way `render --check` catches an edit of a `<task>.md` sidecar (Q-64-generation-architecture sec 2, 5.3).

Realized as a NORMALIZED in-test guard, not a strict byte compare. The self-scaffold pipeline runs `scaffold` then `nix fmt` (prettier, proseWrap=never), so the committed file is `prettier(build_assets output)`, and a raw byte compare would fail on incidental reflow rather than real drift (this composes with Q-57: incidental formatter reflow is not a finding). The guard compares after a `normalize_wrapping` transform that tolerates only prettier's wrapping/whitespace freedoms and passes fenced code verbatim, so it catches real content drift (hand-edit, dropped slot, stale pack source) while ignoring reflow. A precondition fail-safe (`assert_no_unprotected_construct`) asserts every non-fenced line is already in canonical whitespace form, so the guard fails loudly the day the guidance gains an indentation- or whitespace-significant construct (nested list, indented code, multi-space or non-space-whitespace inline code) that `normalize_wrapping` would collapse, before such content could mask drift; hardening `normalize_wrapping` is then required. Note (POC finding): the committed files are currently byte-identical to the raw render (the pack authors single-line paragraphs, so prettier is a no-op today), so the normalization is presently the identity and the precondition holds trivially; both are retained as future-proofing.

Serves Make illegal states unrepresentable (a divergence between the authored template plus generated slots and the committed file becomes a failing check) and Structured data first, project for humans (the committed human view is pinned to a fresh projection of the source).
