### `classification-trivial-rename`: Rename the metrics `Classification::Trivial` variant (SE-2)

Deferred cleanup from the `Q-44` audit (`architecture-audit.explorations/audit-sharp-edges.md`, SE-2). The token `trivial` named two unrelated concepts: a Roadmap step status (a W3-exempt completion) and a `Classification` variant in the intake metrics enum (`Trivial => "trivial"` in `src/metrics.rs`, a per-round risk class). The Roadmap `trivial` status has since been removed (folded into the unified waiver model; it is absent from the clean-slate `StepStatus` enum), so the rename that was previously blocked by the live status collision is now unblocked: rename the metrics `Classification` variant to a distinct token so the two senses no longer share a string. No open decision, a known change. Deferred as an independent naming cleanup.
