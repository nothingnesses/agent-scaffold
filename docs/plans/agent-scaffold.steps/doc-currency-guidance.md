### `doc-currency-guidance`: documentation currency as core cross-cutting workflow guidance (`Q-50`)

Not started. Decided (`Q-50`, human, 2026-07-19). Make DOCUMENTATION CURRENCY a CORE CROSS-CUTTING guidance in the workflow, not an optional module and not a deterministic check alone. Concretely: add a documentation-impact assessment to the PLANNER's duties (when folding a change, the planner identifies which docs and prompts the change makes stale) and to `pack/AGENTS.md` phase 2, plus a doc-currency check at the ACCEPTANCE step (does the shipped change leave any doc or prompt stale?). Reuse the EXISTING roles rather than adding a new role or module: the planner assesses, the implementer updates, the reviewer verifies. Chosen over (a) an opt-in `--module` bundle and (b) a deterministic check alone. This complements, and is a prerequisite for, the workflow-driver's later doc-generation stages (`Q-51`) if those are built: the driver can generate a doc fragment from a machine definition only once staleness is a first-class assessment the workflow already makes. Scope is a pack-doc-and-guidance edit (touches `pack/AGENTS.md`, the planner prompt, and the acceptance step guidance) that goes through the normal implementer plus review loop; sequenced after Inc 6 of `structured-skeleton` (done).
