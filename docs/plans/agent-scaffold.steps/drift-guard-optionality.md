### `drift-guard-optionality`: Drift-guard checks field optionality, not just presence (SE-6)

Deferred cleanup from the `Q-44` audit (`architecture-audit.explorations/audit-sharp-edges.md`, SE-6). The metrics schema drift-guard test (`src/metrics.rs:674`) confirms every validator-accepted enum value appears in `pack/instrument.md`, but it checks a field NAME's presence only, not whether a field documented as required in `instrument.md` is actually required by the parser (or that optional fields agree). So `validate_log()` can accept a record missing a field the docs call mandatory, and the mismatch accumulates silently. Extend the guard to compare required/optional status, not just name presence. No open decision, a known change. Deferred as an independent enforcement-hygiene cleanup.
