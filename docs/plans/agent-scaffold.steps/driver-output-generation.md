### `driver-output-generation`: generate driver outputs at the point of action without drift; harden writer isolation (`Q-51` follow-up)

The human-approved SURGICAL restate-by-generation bundle for the workflow driver (`next`): make its instructions and reminders effective at the point of action (inline content, not a bare pointer) WITHOUT drifting from the canonical docs, by GENERATING the inline content (projection from a single source with a regenerate-and-byte-compare guard) rather than hand-copying it. The full design, the categorization of every driver output, and the per-decision reasoning (D-a through D-e) live in `driver-output-generation.design.md`; this sidecar only names the decided scope. Do not restate the design here.

Decided scope (five surgical changes):

- A shared generated `isolation_policy` fragment (the proven Stage-0b `control_fragment` pattern): author the tier preference order and the standing writer-isolation directive ONCE, substitute it into a new AGENTS.md render slot AND emit it as the driver's writer-state reminder, and byte-guard both against the one fragment. This is also where the planner-isolation clarification (formerly the `agents-worktree-planner-scope` step) is authored, so generation carries it into both AGENTS.md and the driver reminder and the guard holds them together.
- Fix `spawns_writer` to `{ReadyToPlan, AwaitingFixes}`: a real bug. It currently also fires on `AwaitingFirstReview` / `AwaitingReviewers`, which spawn read-only REVIEWERS that need no isolation, contradicting `pack/AGENTS.md` and `orchestrator.md`. This is a prerequisite for the always-on reminder below.
- An always-on writer-isolation reminder in `next`: emit the generated fragment plus the projected resolved tier at the true writer states unconditionally, dropping the `tier == "unknown"` gate.
- Selective principle-text projection: strip the fragile numeric "Principle N" citations from the terse workflow-phase reminders (they become honest originated imperatives), and project a `[[principle]]` `name`/`text` live from the plan source ONLY for the reminder that exists to put a Project Principle in front of the human (the escalate / human-input-contract case), degrading to the originated imperative alone when the referenced principle is absent from the plan.
- An orchestrator-prompt writer-spawn preamble: if it is to be a restated checklist at the spawn point, reuse the SAME generated `isolation_policy` fragment (not a hand-copied checklist); if a pointer is acceptable (the orchestrator loads its own prompt in full), keep the pointer and rely on the fragment and the `spawns_writer` fix.

Deferred, explicitly out of scope for this bundle: role-prompt inlining or summaries (keep the `prompt_path` pointer); generating other prose rules as fragments (promote a rule only when a specific driver state must restate it); all enforcement / machine-gating on the tier (Stage 3+; this bundle keeps isolation output advisory); the typed multi-loop `src/driver/` FSM engine (Stage 2, forward-compatible with this generation strategy).
