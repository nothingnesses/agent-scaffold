### `file-safety-rules`: Git-durability-and-recovery discipline

Decided (`Q-17`, adopted by the human). Add to `pack/AGENTS.md` an always-on file-safety discipline that works on any harness, whether or not it offers isolation; this is the baseline that `agent-isolation` layers a structural upgrade on top of, not a substitute for it. Git is the durability and recovery substrate: the rules keep every writer's damage to a visible, committed-or-recoverable diff. The discipline:

- Clean-tree-before-writer: commit pending work (especially the ledger) before spawning a writer agent, so a writer's kill or misstep leaves only a visible uncommitted diff and never risks already-decided state.
- Commit-before-delete: commit any workflow-managed file (a findings file, the ledger at task close, any transient artifact) before deleting it, so the deletion is a committed deletion recoverable from git history.
- Format only your own files: implementers must NOT run repo-wide formatters (`just fmt` / `nix fmt`) or `git checkout` / `git restore` on files they do not own; they format only the files they changed and leave incidental reformatting to the orchestrator.
- Orchestrator recovery protocol: on any agent kill or interrupt, the orchestrator inspects `git status` and the diff, reverts stray temp artifacts, discards or completes partial work, and confirms a known-good tree before continuing.
- Validation-in-scratch: destructive validations run in a temp directory or worktree, not the live tree.

Motivating incidents (both in the ledger's post-189b7bd notes): an implementer ran repo-wide `just fmt` then `git checkout` on the ledger it did not own, clobbering uncommitted ledger notes; and an implementer was accidentally killed mid-task, which a clean tree and a recovery protocol make safe. Change `pack/AGENTS.md` (a file-safety / durability section) and the role prompts that need it (`pack/prompts/implementer.md` for the format-only-your-own-files and validation-in-scratch rules; `pack/prompts/orchestrator.md` for clean-tree-before-writer, commit-before-delete, and the recovery protocol). Validate by reading the rules across `pack/AGENTS.md` and the prompts and confirming each writer role and the orchestrator carry the rule that applies to it, stated once and consistently.
