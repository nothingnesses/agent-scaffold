### `init-vcs`: Initialise a VCS repository (git by default)

`--vcs <git|none>` (default `git`) initialises an empty version-control repository in the output directory. Git is initialised by default, matching cargo; `--vcs none` skips it. This reverses OQ-K's original opt-in `--init` default (the user chose cargo's on-by-default convention); it is reconciled with safe-on-existing (Principle 3) by never nesting or reinitialising (see below), and `--vcs none` preserves the opt-out. Behaviour:

- Default git, `none` to skip: `--vcs` is a clap value-enum (`Vcs { Git, None }`) defaulting to `Git`; the reserved future extension (jj, hg, and so on) slots in as more variants without changing the default.
- Safe on existing (cargo-style): `inside_git_repo` walks up from the output directory (resolving a relative path against the current directory) looking for a `.git` entry; if the directory is already inside a repository the action is skipped, so scaffolding into a subdirectory of an existing repo never nests a new one, and an existing repo is never reinitialised (Principle 3).
- Mechanism: `run_git_init` shells out to `git init <output-dir>` (no libgit2 or Nix dependency, consistent with `git-url-fetch`). If a repository would be initialised but the `git` binary is not found, the run errors (exit 2) rather than silently skipping; `--vcs none` needs no git.
- Write-safety and idempotency: `init_plan` decides the action from a read only (`InitPlan::Init` / `SkipExists` / `None`), so the preview never writes. The repository line appears first in the plan (`init  git repository`, or `skip (exists)  git repository` when already inside a repo) and `git init` runs only when writing is confirmed (`--write` or the selector's Save), before the assets are dropped. The repository is left empty (no initial commit and no `.gitignore`); committing the scaffolded files stays the user's choice.

Validated by proof-of-concept: a unit test asserts `init_plan`'s decision (default git in a fresh directory -> init, `--vcs none` -> none, a directory that is a repository -> skip, a subdirectory of a repository -> skip so nothing nests) without needing a git binary; and functional runs confirmed that a default run into a fresh directory with `--write` creates `.git`, `--vcs none` writes no repository, scaffolding into a subdirectory of an existing repo skips (no nested repo), and a default run with `git` absent from `PATH` errors (exit 2) before any asset is written. 46 tests pass; clippy `-D warnings` clean. The fallback (raise rather than add a git-library dependency) was not needed.
