### `resume-state-currency-signal`: the minimal Q-59 lifecycle outcome, folds into `Q-58`

What it is: add a currency field (a checkpoint-commit / last-updated marker) to Q-58's structured transient resume-state carrier, and project it in `next` and `status` as "current / N commits stale / tree dirty". This is the whole build scope of the Q-59 decision (human, Option A, 2026-07-23): a currency signal, not a lifecycle state machine.

Why: the one lifecycle failure actually recorded on this repo is a currency problem, the stale resume-anchor and the `next` output bloat (measured 150145 bytes, of which only the 799-byte ACTIVE LOOP block was live). No recorded transition failure exists (no botched resume-without-checkpoint, no work-started-while-paused), so there is no evidence to justify building transition machinery. P6 (ground decisions in evidence, build only what has evidence) is the driver: build the signal the recorded failure calls for, and nothing more. P2 (minimal by default) rules out the command surface and the enum. P1 (prefer the cleaner long-term architecture) directs folding this into Q-58's single structured-transient carrier rather than standing up a second lifecycle mechanism.

Deferred behind an evidence gate: the richer lifecycle state (a `session_state` enum, a W6 transition-legality check, and refuse-while-checkpointed advice in `next` / `status`) waits for a first recorded transition failure, mirroring Q-51's staged-behind-gates precedent. No `checkpoint` / `resume` write-command pair (all three Q-59 explorers rejected it: it jumps the Q-24 guarded-write gate and adds a runtime write path). The state lives in the Q-58 carrier where `next` loads it unconditionally, not the `--instrument`-gated `workflow.jsonl` (which fails P3, safe on existing projects, as a sole home).

Gating: this is blocked on Q-58's structured-transient carrier being designed. Q-58 is itself `open`, gated on the ablation experiment that decides its output content, so this step cannot start until Q-58 resolves and the carrier exists to hang the currency field on.
