### `reviewer-diversity`: Generalise the reviewer/triager diversity rule to models OR harnesses (`Q-36` part a)

Not started. Decided (`Q-36`, human, after the two-explorer cross-harness design pass; design record in `docs/plans/cross-harness-isolation.explorations/q33-q36-{A,B}.md`). A SMALL, always-on core-docs edit: generalise the existing reviewer/triager model-diversity rule from "prefer several reviewers with different lenses, and different models where available" to "different models OR HARNESSES where available", so a user who runs multiple CLIs (for example Claude Code plus another model's CLI) is guided to spread reviewers across harnesses too, minimising the shared blind spots that same-model (and same-harness) reviewers share. Harness-agnostic, so it ships as core content, not a module: a one-clause edit to `pack/AGENTS.md` (the Reviewers role line and the Design-explorations "different lenses or models" line) and to the `pack/user-prompts/review.md` depth-and-lenses cue, then regenerate the self-scaffold. (`pack/prompts/reviewer.md` and `pack/prompts/triager.md`, named as candidate targets during planning, carry no diversity clause to generalise, so the build deliberately left them unchanged: reviewer/triager diversity is an orchestrator-spawning concern, canonical in `pack/AGENTS.md`, one source of truth. The `review.md` cue was the one further user-facing surface the review loop found still framing diversity as models-only, and it was generalised too.) This is the ADOPT-NOW half of the `Q-33` / `Q-36` cross-harness decision; the DEFER half (the `.agents/spawn.toml` launch-map and the container-wrapper shim that would let the orchestrator actually spawn cross-harness agents) is recorded as the deferred growth path in `optional-modules` increment (3). Judgment call (planner): kept as its own small step rather than folded into an existing one, so the Roadmap and the `validate --plan` cross-reference each carry a slug for the always-on rule; it has no code and no dependencies, so it can land at any time.
