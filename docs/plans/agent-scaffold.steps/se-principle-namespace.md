### `se-principle-namespace`: Namespace the two Project Principle citations (SE-1/SE-17)

Deferred cleanup from the `Q-44` architecture audit (`architecture-audit.explorations/audit-sharp-edges.md`, SE-1 and its code-level symptom SE-17). The repo has two independent principle-numbering systems: the Plan Project Principles (P1-P8, in this plan) and the scaffolded AGENTS.md Principles (P1-P22, rendered from `pack/principles.toml`). Code comments cite both with a bare `Principle N` and no namespace prefix (for example `src/plan.rs:11-12` cites the Plan's P5 and AGENTS.md P16 back to back), so a reader cannot tell which system a citation means without cross-referencing both documents. The fix is a citation-hygiene change: prefix each citation with its namespace (for example `Project P5` vs `Scaffold P16`) or cite by id string, at the sites listed in SE-1 (`src/plan.rs`, `src/metrics.rs`, `src/main.rs`, `src/checks.rs`) and the module headers in SE-17. No open decision, a known change. Deferred as an independent naming cleanup the structured-data pivot does not itself resolve.
