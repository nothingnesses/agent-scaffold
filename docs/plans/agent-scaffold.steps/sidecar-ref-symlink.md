### `sidecar-ref-symlink`: Symlink escape in `[meta].sidecars` refs (L1)

Deferred hardening (issue L1). `is_safe_sidecar_ref` (`src/plan/source.rs:489`) is purely LEXICAL: it rejects absolute paths and `..` components, but a task-relative ref that resolves through a SYMLINK can still point outside the plan directory, so render could splice bytes from an out-of-tree file. This is low risk under the current trusted in-repo model (the `.plan.toml` and its sidecars are authored in the same repo), so it is deferred, but it is a prerequisite before `git-url-fetch` (the deferred git-URL template fetch) ever renders an UNTRUSTED `plan.toml`: at that point a symlink-aware check (resolve the real path and confirm it stays under the plan directory) is required. No open decision on the fix; the only open point is WHEN, gated on untrusted-source rendering. Deferred.
