### `triager-independence`: The triager is always a separate agent

Decided by the human (`Q-13`): ALWAYS a separate triager, NEVER collapsed. Amend `pack/AGENTS.md`'s role-separation rules so the triager is always a separate agent, independent of BOTH the agent that produced the artifact under review AND the orchestrator, for every review round including trivial ones. The orchestrator drives the loop and owns convergence and cost, so it is biased toward dismissing findings to converge; letting it triage would let that bias decide which findings count.

Changes to `pack/AGENTS.md`:

- Narrow the "Match the ceremony to the stakes: collapse roles for a trivial change" rule so it explicitly does NOT permit collapsing the triager: the triager stays separate even for a trivial or low-risk review round.
- Update the Triager role description (today it states only "The triager must not be the agent that produced the artifact under review") to add the orchestrator-not-triager independence alongside the existing producer-not-triager rule.

Align `pack/prompts/triager.md` and `pack/prompts/orchestrator.md` with the strengthened rule where they name who triages.
