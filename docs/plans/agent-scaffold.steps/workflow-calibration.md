### `workflow-calibration`: Calibrate the workflow's provisional round constants

Not started (deferred). The scaffolded workflow's round constants and the second-triager guard were introduced after a design review (see the `## [Unreleased]` entry in `CHANGELOG.md`) as provisional defaults chosen to be safe rather than measured: the required consecutive clean rounds (one for a trivial or low-risk artifact, two for a risky one) and the total-round cap (five). This step calibrates them against data from real use. This is deliberately deferred: the fixes it revisits were split into cheap no-regret changes applied now (durable ledger, termination caps, acceptance triager, dismissed-high-severity re-check) and the one recurring-cost change (consecutive clean rounds), whose default is provisional pending this data, so nothing here blocks earlier work.

Data to gather. Per review round: the artifact and phase (plan review, work review, or acceptance), whether the artifact changed since the previous round, the round outcome (clean, or new valid findings), the count and severities of valid findings, and the running consecutive-clean count. Per escalation: whether the human made a real decision or just resumed (useful versus friction). Per dismissed high-severity finding: whether the second triager upheld or overturned the dismissal. Per human interrupt: the trivial or non-trivial classification and whether a trivial call later had to be re-planned (a misclassification). The key signal for the required-clean-rounds default is whether a round following a clean round finds a triager-valid issue on an unchanged artifact, which is evidence that one clean round is noisy.

Collection method. For agent-scaffold's own tuning, the round data is recorded to the opt-in JSONL round log (`docs/metrics/workflow.jsonl`), which carries the structured per-round outcomes (including `risk_class` and reviewer attribution) for this reason; the ledger's round-summary table was removed by `round-log-core` (`Q-34`), so structured round data lives only in the round log now. Downstream users get collection through the opt-in `--instrument` flag, not baked into the pack. Trigger: revisit the defaults once the first several real tasks have produced data. Evidence-grounding: the adjusted defaults must be backed by the logged round outcomes, not re-guessed; if the data is inconclusive, keep the safe defaults and record that.

Methodology design inputs (recorded 2026-07-18, from a human-driven Socratic thread during `optional-modules` increment 2; not yet decided, to be worked out in this step's design pass). These sharpen "calibrate against data" into concrete, statistically-grounded techniques, and share ONE prerequisite:

- Per-finding identity (the enabling schema change). To compute reviewer overlap, marginal yield, or any inter-rater statistic, the round record must LINK the same finding across reviewers (a stable finding id / dedup key), which it does not today (it records per-reviewer counts only). This is the same gap `reviewer-harness-field` deferred here (the unique/marginal-valid count). It unlocks every item below, so it is the first design question.
- Statistical rigor, method matched to the data (not a blanket). Inter-rater methods like Fleiss's / weighted Cohen's kappa FIT a few specific measurements: severity-rating agreement (ordinal, weighted kappa on co-rated findings) and triager/backstop reliability (the `dismissal_recheck` upheld/overturned rate is a two-rater agreement, though zero such records exist yet). They do NOT fit reviewer finding-GENERATION (reviewers generate, not classify a shared item set), where overlap/set-similarity (Jaccard, unique-valid count) is correct. The clean-round constant is really a CENSORING problem (we stop at the streak, so never observe what another round would find), better served by a sequential/survival framing than by agreement stats. Honest caveats to bake in: report every estimate WITH a confidence interval (n is tiny), and remember LLM raters share correlated blind spots (same-family agreement overstates reliability, cross-harness agreement is the meaningful signal, which is why `reviewer-diversity` matters).
- How many reviewers / which combinations (deciding beyond the current 2). Two levers to keep distinct: clean-ROUNDS (already risk-scaled 1/2) versus reviewers-PER-round (currently a soft "prefer several"). Decide reviewer count/combination by risk x count-of-distinct-failure-modes x independence (lens < model < harness) x cost; default 2 diverse as the floor; scale to 3+ when a genuinely distinct third failure mode exists or measured marginal unique-valid yield stays high; add a new HARNESS only with a PoC + measurement (`Q-33`/`Q-36`). The data-driven form: keep adding a reviewer while its marginal unique-valid yield exceeds a threshold (the reviewer-count analogue of loop-until-clean), which needs the per-finding-identity schema above. A smaller shippable slice: tighten the pack rule from "prefer several" to an explicit risk-scaled reviewer-count-and-diversity guideline (mirroring how clean-rounds scale with risk).
- Making risk more objective, and updating it as findings are addressed. Grade each judgement into mechanical / intersubjective-reliable / irreducibly-subjective, and push the boundary toward mechanical (Principle 6). For risk specifically: turn the criteria into a scored rubric of near-yes/no factors (touches auth/crypto/secrets? money? persistent data/migrations? a widely-depended-on interface? an irreversible or out-of-scope operation?), record it once (auditable), and make it FALSIFIABLE by scoring `risk_class` against outcomes (do `low_risk` artifacts actually escape fewer late high-severity findings?). Risk is fixed within a loop by design (anti-drift, so the orchestrator cannot downgrade to converge); a downgrade requires human authority (the escalation contract). A more principled convergence rule could key on the SEVERITY TRAJECTORY of findings (converge after N consecutive rounds with no finding above severity X) rather than a binary clean/not-clean count, which would let "risk falls as the dangerous findings are closed" become an objective, logged criterion, exactly the judgement used to accept `optional-modules` 2c-ii at one clean round (see `Q-40`).
