## Findings

### F1. Decided spec item omitted: the concrete preflight container-tier detection cue
Severity: low
Evidence: `pack/isolation-guidance.md:35` (the only preflight sentence) vs `docs/plans/agent-scaffold.md:530`.
The settled `Q-41` file shape says the partial supplies, among the container-tier wiring, "a preflight container-tier detection cue (`which ab` + the configured image present)". The guidance mentions preflight only as "The orchestrator states the resolved isolation tier at preflight, so you know before a run whether a writer will run in a container, a worktree, or under the file-safety fallback." It gives no concrete detection cue (no `which ab`, no "configured image present" check). This is a genuine deviation from the decided spec. Impact is small because the AGENTS.md Preflight rule (`AGENTS.md:98`) already owns "detects and states the writer-isolation modes and tools actually available", so the concrete cue is a convenience the guidance chose to defer to that rule rather than restate. A user is not misled, but the specced item is not present. Flagging for the triager to decide whether the spec item is mandatory or the deferral is acceptable.

### F2. Terminology overlap risk: agent-box `--git` "worktree mode" vs the tier-2 "worktree isolation"
Severity: low
Evidence: `pack/isolation-guidance.md:27-35`.
The guidance frames agent-box presence as the container tier (para 1, correct), then the "Run a writer under isolation" section instructs `ab new ... --git` / `ab spawn --git`, calling it "the sandboxed worktree mode so a writer agent sees only committed files". agent-box's `--git` mode is a container run that additionally uses a git worktree for file visibility, so it is still tier-1 container isolation. But AGENTS.md tier 2 is literally named "Worktree isolation ... a git worktree" (`AGENTS.md:82`), a bare worktree with no container. A reader could conflate agent-box `--git` mode (container + worktree) with the tier-2 bare worktree. Para 1 keeps the tiers distinct, so this is a mild clarity risk, not a contradiction. No fix required for correctness; noting in case the triager wants a one-clause disambiguation.
