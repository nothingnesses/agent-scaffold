# Clarifying questions

First, read `AGENTS.md` (and any existing plan for this task under `docs/plans/`),
so your questions and recommendations are aligned with the project's current
principles rather than assumed ones.

Then, before you start, raise any clarifying questions you have about this task for
the human to answer, so the task is confirmed before any code is written. Where you
would otherwise make an assumption, raise it instead. For each, give your
recommendation and the reasoning behind it, so the human can confirm or override
rather than reconstruct your thinking. If nothing is unclear, say so and state the
assumptions you are proceeding on.

The human is the decider here. If you are a sub-agent without a direct channel to
the human, return your questions to the orchestrator, which surfaces them and
relays the human's answers.
