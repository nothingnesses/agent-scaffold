# Open-questions gate

First, read `AGENTS.md` so you are working from the project's current principles.

Before proceeding to the next steps of implementation, are there any open
questions, decisions, issues, or blockers that should be addressed first?

If so, add them to the plan's "Open Questions, Decisions, Issues and Blockers"
section, and for each present the decision per the human-input contract in
`AGENTS.md`, for the human to review and choose from. Prefer the approaches most
consistent with the project's principles.

Otherwise, state that none are open and proceed with the next steps.

The human decides which options to take. If you are a sub-agent without a direct
channel to the human, return the open items to the orchestrator, which relays them
to the human and returns the decisions.
